# Contributing to BeeHappy

First off, thank you for considering contributing to BeeHappy. It's people like you that make BeeHappy such a great app.

## Where do I go from here?

If you've noticed a bug or have a question, [make one](https://github.com/yourusername/your-project-name/issues/new)! It's generally best if you get confirmation of your bug or approval for your feature request this way before starting to code.

## Fork & create a branch

If this is something you think you can fix, then [fork BeeHappy](https://help.github.com/articles/fork-a-repo) and create a branch with a descriptive name.

A good branch name would be (where issue #325 is the ticket you're working on):

```bash
git checkout -b 325-add-japanese-translations
```

## Get the code

The steps below will walk you through how to get the code on your machine.

```bash
# Clone your fork to your local machine
git clone git@github.com:YOUR-USERNAME/BeeHappy.git
```

## Code Guidelines

- Make sure you follow the coding style that is used in the project.
- If you're creating a new file, make sure to include the appropriate license header at the top of the file.

## Submitting the Pull Request

Once you're satisfied with the changes, and you've adhered to the code guidelines, go ahead and create a [Pull Request](https://help.github.com/articles/creating-a-pull-request).

---

Thanks for contributing!
